# pZero Design System

The single source of truth for every visual and interactive decision in pZero (Phoenix OS). All UI surfaces — web, mobile, dashboard — must use these tokens and follow these conventions.

**Stack:** Next.js App Router · shadcn/ui · Tailwind CSS · Geist Sans / Geist Mono

---

## 1. Design Principles

| Principle | Meaning |
|-----------|---------|
| **Fewer clicks, more clarity** | Every extra interaction is a failure. Optimise for the fastest path to insight and action. |
| **Trust through polish** | Rough edges destroy credibility with early adopters. Finish every state. |
| **Accessibility = inclusion** | WCAG 2.1 AA is a floor, not a ceiling. |
| **Consistency over novelty** | A coherent system beats clever one-offs every time. |
| **Dark mode first** | pZero is an AI-native developer tool — dark surfaces reduce fatigue and signal craft. |

---

## 2. Color Tokens

### Base palette

pZero uses **zinc** as the neutral scale, with **indigo** as the single accent. All values are Tailwind CSS class names. For raw HSL values, use the CSS variables generated by shadcn/ui init.

| Role | Token | Tailwind class | Usage |
|------|-------|----------------|-------|
| **Background / base** | `--background` | `bg-zinc-950` | Page background |
| **Surface 1** | `--card` | `bg-zinc-900` | Cards, panels |
| **Surface 2** | `--muted` | `bg-zinc-800` | Secondary surfaces, hover states |
| **Border** | `--border` | `border-zinc-800` | All dividers and outlines |
| **Border (subtle)** | `--border-subtle` | `border-zinc-700` | Inner dividers |
| **Text / primary** | `--foreground` | `text-zinc-50` | Headings, primary body |
| **Text / muted** | `--muted-foreground` | `text-zinc-400` | Labels, metadata, placeholders |
| **Text / disabled** | `--disabled-foreground` | `text-zinc-600` | Disabled states |
| **Accent** | `--primary` | `bg-indigo-500` | CTAs, active states, links |
| **Accent hover** | `--primary-hover` | `bg-indigo-400` | Hover on accent |
| **Accent foreground** | `--primary-foreground` | `text-white` | Text on accent backgrounds |
| **Destructive** | `--destructive` | `bg-red-600` | Delete, irreversible actions |
| **Warning** | `--warning` | `bg-amber-500` | Warnings, degraded states |
| **Success** | `--success` | `bg-emerald-500` | Confirmations, completed states |

### Rules

- **Do not** introduce new brand colors without UX Lead sign-off.
- **No rainbow accents.** One accent color. Use semantic colors (destructive, warning, success) only for feedback states.
- **No heavy gradients or glassmorphism** unless a specific data-viz use-case is approved.
- Minimum contrast ratio: **4.5:1** for normal text, **3:1** for large text and UI components (WCAG 2.1 AA). See [../docs/design/accessibility.md](../docs/design/accessibility.md).

## 3. Typography

### Font families

| Family | Usage | CSS variable |
|--------|-------|-------------|
| **Geist Sans** | All interface text | `--font-geist-sans` |
| **Geist Mono** | Code, metrics, IDs, timestamps, terminal output | `--font-geist-mono` |

Install via `next/font` (add `geist` to dependencies):

```tsx
// app/layout.tsx
import { GeistSans } from 'geist/font/sans'
import { GeistMono } from 'geist/font/mono'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={`${GeistSans.variable} ${GeistMono.variable} dark`}>
      <body>{children}</body>
    </html>
  )
}
```

```ts
// tailwind.config.ts — extend theme to use CSS variables
theme: {
  extend: {
    fontFamily: {
      sans: ['var(--font-geist-sans)'],
      mono: ['var(--font-geist-mono)'],
    },
  },
},
```

Use `font-sans` for interface text and `font-mono` for code/metrics in Tailwind classes.

### Type scale

| Step | Class | Size | Weight | Line height | Usage |
|------|-------|------|--------|-------------|-------|
| `display` | `text-3xl font-semibold` | 30px | 600 | 1.2 | Page titles |
| `heading-1` | `text-2xl font-semibold` | 24px | 600 | 1.25 | Section headings |
| `heading-2` | `text-xl font-medium` | 20px | 500 | 1.3 | Sub-section headings |
| `heading-3` | `text-base font-medium` | 16px | 500 | 1.4 | Card titles, list group headers |
| `body` | `text-sm` | 14px | 400 | 1.5 | Default body text |
| `caption` | `text-xs text-zinc-400` | 12px | 400 | 1.5 | Helper text, timestamps, metadata |
| `code` | `font-mono text-sm` | 14px | 400 | 1.6 | Inline code, IDs |
| `code-block` | `font-mono text-xs` | 12px | 400 | 1.6 | Multi-line code blocks |

### Rules

- **Never mix font families** in the same element.
- Use `text-zinc-50` for headings, `text-zinc-300` for body, `text-zinc-400` for captions.
- Avoid `font-bold` (700) in UI — prefer `font-semibold` (600) for emphasis.

## 4. Spacing

pZero uses Tailwind's default 4px base grid. All spacing must be a multiple of 4px.

| Token | px | Tailwind | Usage |
|-------|-----|----------|-------|
| `spacing-1` | 4px | `p-1 / m-1` | Tight inline gaps |
| `spacing-2` | 8px | `p-2 / m-2` | Icon padding, badge padding |
| `spacing-3` | 12px | `p-3 / m-3` | Small card padding |
| `spacing-4` | 16px | `p-4 / m-4` | Default card padding, list item padding |
| `spacing-6` | 24px | `p-6 / m-6` | Section padding |
| `spacing-8` | 32px | `p-8 / m-8` | Page section gaps |
| `spacing-12` | 48px | `p-12 / m-12` | Large section separators |
| `spacing-16` | 64px | `p-16 / m-16` | Hero / landing page spacing |

## 5. Elevation & Shadows

Dark surfaces rely on **borders** and **background lightness** for depth — not box shadows. Shadows are reserved for floating elements (dropdowns, tooltips, modals).

| Level | Class | Usage |
|-------|-------|-------|
| `0` | `border border-zinc-800` | Cards, flat panels — border defines the edge |
| `1` | `shadow-sm` | Popovers, dropdowns |
| `2` | `shadow-md` | Modals, dialogs |
| `3` | `shadow-lg` | Toasts, command palette |

---

## 6. Border Radius

| Token | Class | Usage |
|-------|-------|-------|
| `radius-sm` | `rounded` | Buttons, badges, inputs |
| `radius-md` | `rounded-md` | Cards, panels, dropdowns |
| `radius-lg` | `rounded-lg` | Modal dialogs, sheets |
| `radius-full` | `rounded-full` | Avatars, status dots, pill badges |

**Rule:** Do not mix radii in the same component. Cards are always `rounded-md`. Buttons always `rounded`. Modals always `rounded-lg`.

## 7. Component Conventions

### Structural layout patterns

| Pattern | When to use |
|---------|-------------|
| `Tabs + Card + Form` | Settings pages |
| `Card + Table + Filters` | Dashboard list views |
| `Sheet` | Mobile navigation, detail drawers |
| `AlertDialog` | Destructive confirmations |
| `Command` | Global search / command palette |
| `Popover` | Contextual actions, quick edits |

### Interactive states (all components must implement)

| State | Visual treatment |
|-------|-----------------|
| Default | Base token |
| Hover | `bg-zinc-800` overlay or accent shift |
| Focus | `ring-2 ring-indigo-500 ring-offset-2 ring-offset-zinc-950` |
| Active / pressed | Slight scale or background darken |
| Disabled | `opacity-50 cursor-not-allowed` |
| Loading | Skeleton or spinner in-place; never blank |
| Empty | Illustrated or described empty state |
| Error | Destructive color + icon + message |

### Focus ring (critical for keyboard navigation)

Every interactive element must show a visible focus ring. Use this exact class composition:

```
focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-indigo-500 focus-visible:ring-offset-2 focus-visible:ring-offset-zinc-950
```

Do **not** use `outline-none` alone without replacing the focus indicator.

## 8. Iconography

Use **Lucide React** exclusively (`lucide-react`). Do not mix icon libraries.

| Size | Class | Usage |
|------|-------|-------|
| `sm` | `h-3 w-3` | Inline status dots |
| `md` | `h-4 w-4` | Button icons, list row icons (default) |
| `lg` | `h-5 w-5` | Navigation icons |
| `xl` | `h-6 w-6` | Empty state icons |

Icons must have `aria-hidden="true"` when decorative. When standalone (no visible label), add `aria-label` or a `<VisuallyHidden>` sibling.

---

## 9. Animation & Motion

- Use `transition-colors duration-150` for color state changes.
- Use `transition-opacity duration-200` for fade in/out.
- Respect `prefers-reduced-motion`: wrap all animations in `@media (prefers-reduced-motion: no-preference)` or use Tailwind's `motion-safe:` variant.
- Avoid layout-shift animations that change element position on hover.

---

## 10. Dark Mode

Dark mode is **always on** for pZero. Do not implement a light/dark toggle unless explicitly requested.

- Set `className="dark"` on `<html>` in `app/layout.tsx`.
- All shadcn/ui components respect this automatically via CSS variables.
- Test all new components on a dark background. Never only test on white.
